本文记录最近帮别人修理一台老笔记本的琐事
部件 参数
笔记本电脑 东芝 Satelite L675
系统 Windows 7 SP1 旗舰版
CPU AMD（忘了型号了，回头补上）
固态硬盘 ThinkLife ST800系列 SATA3-512G

异常状况
#

常常由弹窗错误提示。异常卡顿。经常蓝屏，比如 Chrome 浏览器打开超过 4、5 个标签页，打开一些 Office 文档，复制、剪切文件多时，蓝屏代码各有不同。文末附 几个样例

修改过的系统设置和各磁盘文件，磁盘格式化，重启后又恢复原样。

在 Linux U 盘安装盘系统中，用 fdisk 重新分区、删除分区、重建分区表等操作后，均恢复原样。通过 mv 将大量文件从 D 盘 挪至 E 盘时，偶尔出现不同的失败情况。

在 PE 系统中，重新分区、删除分区、重建分区表等操作再刷新后，均恢复原样。用过的各种磁盘分区工具，如傲梅分区助手、DiskGenius、Max 工具箱等。

尝试使用 Windows 原装镜像重装系统时，提示 C 盘安装系统后会磁盘损坏（或者类似描述，记不清了）。

思路与解决
#

起初从蓝屏代码推测，说不定是硬盘或显卡驱动的问题。后来考虑可能是内存问题，当占用过多时会崩溃，此时未保存妥帖的内容由某种位置原因回滚。再后来，通过尝试，发现各磁盘文件的修改都会恢复。

此时考虑，多数异常情况的共同特点是「磁盘写入行为异常」，而磁盘是计算机的下属设备，数据由磁盘主控管理，因此推测是磁盘损坏导致无法修改，只能读取。于是考虑更换硬盘。

更换硬盘后，重装 Windows 7 系统，一切正常了。后续，驱动精灵打上驱动，火绒扫描漏洞后参考 Windows 7更新（补丁）整理 挑选了几个补丁打上，装上其他软件，一切正常。

吐槽
#

电脑真老啊，电源插头插不上现在的国标插座。印象里听说，很久以前咱这儿流通着各国的电器，各有各的标准，也不知道这台笔记本是不是外国货。

应该是因为 BIOS 太老了，冰封 v17 PE 盘无法引导，自动进入 grub 命令行。不熟 grub，尝试扫描 PE 的引导，未成功。

换成 微 PE 工具箱 才成功，而撰本文时才发现，网上有好多叫微 PE 的啊……好像我用的是正主，真走运……
